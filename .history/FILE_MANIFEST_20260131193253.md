### Backend Core Files

#### server/src/index.js
- **Purpose**: Express server entry point
- **Exports**: Express app and Socket.io instance
- **Key Features**: 
  - MongoDB connection
  - Middleware setup (CORS, JSON parsing)
  - Route registration
  - Socket.io initialization
  - Health check endpoint

#### Models (server/src/models/)
- **User.js**: User authentication and profile data
  - Fields: name, email, password, role, teamId, isAnonymous, lastActive
  - Indexes: email (unique)
  
- **Team.js**: Team management
  - Fields: name, companyId, managerId, members, description
  - References: User (manager and members)
  
- **Kudos.js**: Kudos/appreciation messages
  - Fields: fromUserId, toUserId, teamId, message, isAnonymous
  - References: User (from/to), Team
  
- **Pulse.js**: Mood/workload check-ins
  - Fields: userId, teamId, status (Light/Good/Heavy), notes
  - References: User, Team
  
- **Question.js**: Q&A session questions
  - Fields: sessionId, teamId, userId, content, upvotes, upvoterIds, isAnswered, answer
  - References: User, Team

#### Routes (server/src/routes/)
- **auth.js**: Authentication endpoints
  - POST /api/auth/register - Register new user
  - POST /api/auth/login - User login
  - GET /api/auth/me - Get current user
  
- **kudos.js**: Kudos management
  - POST /api/kudos/submit - Submit kudos
  - GET /api/kudos/team/:teamId - Get team kudos
  
- **pulse.js**: Pulse check management
  - POST /api/pulse/submit - Submit pulse
  - GET /api/pulse/team/:teamId/aggregated - Get aggregated data
  
- **questions.js**: Q&A management
  - POST /api/questions/submit - Submit question
  - POST /api/questions/:id/upvote - Upvote question
  - GET /api/questions/session/:sessionId - Get session questions
  - POST /api/questions/:id/answer - Answer question
  
- **teams.js**: Team management
  - POST /api/teams/create - Create team
  - GET /api/teams/:teamId - Get team details
  - GET /api/teams/user/my-teams - Get user's teams

#### server/src/middleware/auth.js
- **auth**: JWT verification middleware
  - Extracts token from Authorization header
  - Verifies token signature
  - Sets req.userId, req.userRole, req.teamId
  
- **authorizeRole**: Role-based access control
  - Checks if user has required role
  - Supports multiple roles

#### server/src/services/socketService.js
- **initializeSocketHandlers**: Socket.io event handlers
  - join_room: User joins team room
  - new_kudos: Broadcast kudos to team
  - pulse_submitted: Broadcast pulse update
  - new_question: Broadcast new Q&A question
  - question_upvote: Broadcast upvote update
  - disconnect: User disconnects

#### server/seed.js
- **Purpose**: Populate database with test data
- **Creates**:
  - 4 test users (Manager, 2 Employees, Admin)
  - 1 test team
  - 2 sample kudos
  - 4 sample pulse checks
  - 2 sample Q&A questions
- **Usage**: `node seed.js`
